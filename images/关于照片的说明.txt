照片管理说明

当前文件夹：

- images/cover/：封面和生平纪念区使用的肖像照片
- images/album/life/：生活照片
- images/album/family/：家庭照片
- images/album/group/：亲友合影
- images/album/old/：旧照

当前命名：

- images/cover/portrait.jpg：封面和生平纪念区的肖像照片
- images/album/life/album-01.jpg 等：生活类相册照片
- images/album/family/album-05.jpg 等：家庭类相册照片
- images/album/group/album-03.jpg 等：合影类相册照片
- images/album/old/album-11.jpg 等：旧照类相册照片

新增照片时建议继续使用这个格式：

- 生活照片：images/album/life/life-01.jpg
- 家庭照片：images/album/family/family-01.jpg
- 合影照片：images/album/group/group-01.jpg
- 旧照：images/album/old/old-01.jpg

然后打开网站根目录里的 album-data.js，按已有格式增加一段照片信息：

{
    src: "images/album/family/family-01.jpg",
    title: "照片标题",
    year: "1998",
    category: "家庭",
    caption: "这张照片的简短说明。"
}

常用分类可以写：家庭、生活、旧照、工作、合影。

注意事项：

- 文件名、src 路径、title、year、category、caption 都要保留英文双引号。
- 每一条照片记录之间要用英文逗号分隔。
- 相册页面会按照片原始长宽比显示，不会强行拉伸或裁切。
- 如果想更换封面照片，把新照片命名为 portrait.jpg，并替换 images/cover/portrait.jpg。

管理员页面：

- 打开网站根目录中的 admin.html，可以编辑网站文字和上传临时照片。
- 管理员页面保存的数据会先存到当前浏览器中。
- 如果要长期发布到互联网，请在 admin.html 中导出备份 JSON 或导出 JS 文件，再把导出的 site-data.js / album-data.js 放回网站目录。
- 纯静态网站没有真正的服务器后台，因此 admin.html 不是安全的在线后台；发布正式网站时可以不上传 admin.html，或只在本机使用它整理内容。
